IssGo 是一个运行在本地的 AI Agent 命令行工具。你只需描述「做什么」，Agent 自动分解任务、调用工具执行、汇总结果返回。

Features

为什么选择 IssGo？

不仅仅是 CLI 工具 — IssGo 是一个完整的 AI Agent 运行时，把 LLM 的能力直接带到你的终端。

🗣️

自然语言驱动

不需要记命令。用中文或英文描述你想做什么，Agent 自动理解意图、规划步骤、执行操作。

🔄

ReAct 推理循环

Reason → Act → Observe，Agent 在每一步都先思考再行动，观察结果后决定下一步，直到任务完成。

🧰

7 大内置工具

文件操作、Shell 命令、HTTP 请求、无头浏览器、Git 操作、代码搜索、插件系统 — 覆盖绝大多数自动化场景。

🔌

多 LLM 提供商

DeepSeek、OpenAI、Ollama、Groq、vLLM……兼容所有 OpenAI API 格式的服务，配置文件一键切换。

🪞

自反思 (Reflector)

Agent 在执行后会自我反思：当前方案是否有效？是否需要重新规划？避免陷入无效循环。

🛡️

安全沙箱

静态分析 + LLM 动态审查双重保护。危险命令（rm -rf /、fork bomb 等）自动拦截，可选人工确认。

Commands

命令参考

IssGo 提供 7 个核心命令，覆盖初始化、执行、对话、监听、配置、服务和版本。

issgo init

在当前目录生成 .issgo.yaml 配置文件。使用 --force 或 -f 强制覆盖已有配置。

issgo run

执行 AI 任务。任务描述可以是任意自然语言。支持 --verbose / -v 输出详细日志。

示例：issgo run "审查 cmd/ 目录下的代码，检查错误处理是否完整"

issgo chat

启动交互式对话模式。保持对话历史，支持流式输出。适合多轮复杂任务和探索性操作。

issgo watch

递归监听目录变化，变化时自动触发 AI 任务。--on-change / -c 指定任务，--debounce 设置防抖（默认 500ms）。

示例：issgo watch . --on-change "运行 go test ./... 并修复失败的测试"

issgo config

显示或编辑当前配置。支持 --profile / -p 快速切换 LLM 提供商配置。

issgo serve

启动 HTTP API 服务器（默认端口 8420）。支持 REST + SSE 流式传输，可绑定到任意地址。

示例：issgo serve --host 0.0.0.0 --port 8080

issgo version

显示版本号、构建时间和 Git 提交哈希。版本号通过 -ldflags 在构建时注入。

Configuration

配置参考

YAML 配置文件 + 环境变量覆盖，支持多 Profile 快速切换 LLM 提供商。

.issgo.yaml 完整配置

.issgo.yaml 
llm:


  provider: deepseek          # deepseek | openai | custom


  model: deepseek-chat        # 模型名称


  api_key: ""                 # 留空则从环境变量读取


  base_url: https://api.deepseek.com


  temperature: 0.7


  max_tokens: 4096


  timeout_secs: 120


  retry_count: 3





tools:


  shell: true


  file: true


  web: true


  browser: false            # 需要 Chrome


  git: true


  search: true


  plugins: false


  plugins_dir: ~/.issgo/plugins





agent:


  max_steps: 30             # 单次最大工具调用次数


  allow_approve: true      # 危险操作前确认


  verbose: false


  streaming: true


  reflector: true          # 自反思模块


  max_retries: 3


  session_dir: ~/.issgo/sessions


  max_session_age: 168h





server:


  enabled: false


  host: 127.0.0.1


  port: 8420


  ws: true                 # WebSocket 支持


  auth_token: ""          # API 认证令牌

API Server

HTTP API 接口

通过 issgo serve 启动 HTTP API 服务器，支持 REST + SSE 流式传输，可集成到任何应用中。

方法 路径 描述 请求体
GET /api/v1/health 健康检查，返回版本号 —
POST /api/v1/run 执行 AI 任务，同步返回结果 {"task": "你的任务描述"}
POST /api/v1/stream 执行 AI 任务，SSE 流式传输 {"task": "你的任务描述"}
GET /api/v1/tools 列出所有可用工具 —

快速示例

curl 调用 API 
# 启动服务器


issgo serve --port 8420





# 健康检查


curl http://localhost:8420/api/v1/health


# → {"status":"ok","version":"v2026.06.30"}





# 执行任务


curl -X POST http://localhost:8420/api/v1/run \


  -H "Content-Type: application/json" \


  -d '{"task":"统计项目代码行数"}'


# → {"success":true,"result":"..."}





# SSE 流式传输


curl -X POST http://localhost:8420/api/v1/stream \


  -H "Content-Type: application/json" \


  -d '{"task":"列出所有 Go 文件"}'





# 带认证令牌


issgo serve --port 8420


# 在 .issgo.yaml 中设置 server.auth_token


curl -H "Authorization: Bearer your-token" \


  http://localhost:8420/api/v1/run

FAQ

常见问题

如何安全地存储 API Key？
推荐使用环境变量 export ISSGO_LLM_API_KEY="sk-xxxxxxxx"，添加到 ~/.bashrc 或 ~/.zshrc。也可以放在 .issgo.yaml 中，但注意不要将该文件提交到 Git 仓库。
max_steps 应该设置多少？
默认 30 步对大多数任务足够。简单任务通常 5-10 步即可完成。复杂的多阶段任务可以调大到 50。设置太小可能导致任务未完成就被截断。
shell 工具有哪些安全限制？
两层安全机制：① 静态分析 — 自动拦截 rm -rf /、fork bomb、dd、mkfs 等危险模式；② LLM 动态审查 — 对于警告级别命令（如 rm、git push --force 等），会请求 LLM 评估是否安全。同时有 120s 超时保护。建议开启 allow_approve: true 在执行危险命令前请求确认。
支持哪些操作系统？
Linux、macOS、Windows（通过 Git Bash 或 WSL）。file 和 shell 工具在所有平台可用；browser 工具需要额外安装 Chrome/Chromium。
可以使用本地模型吗？
可以。运行 Ollama 或其他兼容服务后，将配置指向本地地址即可：base_url: http://localhost:11434/v1，model 设置为 qwen2.5:14b 等。注意本地模型需要支持 Function Calling。
如何调试 LLM 调用过程？
配置 agent.verbose: true 或加 -v 参数：issgo run "任务" -v。这会在终端打印完整 Prompt、LLM 响应和 Tool 调用详情。
Reflector 是什么？需要开启吗？
Reflector（自反思模块）在 Agent 完成任务后分析执行过程，判断是否需要重新规划（REPLAN）、继续执行（CONTINUE）、已完成（COMPLETE）或阻塞（BLOCKED）。建议开启，可有效避免陷入无效循环。默认开启。
Session 是如何工作的？
每次对话自动保存为 JSON 文件到 ~/.issgo/sessions/，包含任务描述、完整对话历史、状态和最终结果。Session 默认保留 168 小时（7 天），过期自动清理。可用于恢复中断的任务或审计历史操作。
